Engineering Philosophy

I believe in building understanding from the ground up. Modern development often treats infrastructure as magic, I want to understand the magic. That means:

  • Starting with fundamentals: TCP before HTTP frameworks, sockets before Express
  • Reading source code: Node.js internals, libuv, V8, understanding what happens when I call a function
  • Building what exists: Reimplementing tools to understand their constraints and design decisions
  • Documentation over tutorials: Primary sources teach you to think; tutorials teach you to copy

Projects

HTTP from Scratch in progress

A ground-up HTTP/1.1 server implementation using raw TCP sockets in Node.js (migrating to TypeScript). No frameworks, no http module, just the net module and RFC specifications. Handles request parsing, response formatting, routing, and connection management at the protocol level.

This project taught me how web servers actually work: socket management, HTTP message parsing, connection handling, and the difference between what frameworks give you and what the protocol requires. It's the foundation for understanding modern web infrastructure.
